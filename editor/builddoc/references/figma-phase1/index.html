Ref 001: Simple Rectangles Must Feel Simple

Figma canvas selection
Full-canvas reference: one selected rounded rectangle, clean bounds, compact inspector, and a canvas that stays visually primary.

What matters

  • A plain rounded rectangle is trivial to create and edit.
  • The canvas stays visually dominant while the right rail stays compact.
  • The selected object shows clean bounds and direct handles without UI noise.
  • The object itself feels primary, not the panel explaining the object.

Implication For Stygian

  • Rectangle placement cannot feel like entering a weird sub-mode maze.
  • Selection handles must be obvious, stable, and pleasant to hit.
  • Per-corner radius belongs in an immediate, compact edit path.
  • The viewport needs to feel like the main place where editing happens.

Ref 002: Right Rail Compression Without Losing Power

Inspector mechanics
Inspector reference: dense, compact, and still readable. This is the right kind of compression.

What matters

  • The inspector is grouped by editing intent, not by random implementation data.
  • Per-corner radius is surfaced directly, not hidden behind awkward custom logic.
  • Sections like Fill, Stroke, Effects, and Export are collapsible and predictable.
  • Numeric editing feels native and calm, not bulky.

Implication For Stygian

  • Inspector needs a true section system, not mixed cards with unrelated controls.
  • Transform must become a concise top section: X, Y, W, H, rotation.
  • Appearance must expose corner model, fill, stroke, and effects cleanly.
  • Panel density should feel deliberate, not squeezed and amateurish.

Ref 003: Shape Access Should Be One Gesture Away

Toolbar + primitive menu
Tool access reference: one quiet strip, one compact menu, and fast primitive selection without mode soup.

What matters

  • Shape discovery is centralized and visually compact.
  • Hotkeys reinforce speed without becoming mandatory for basic work.
  • The menu sits near the tool cluster where the user already is.
  • Tool identity stays stable even when a submenu exists.

Implication For Stygian

  • The current top-strip placement model needs to be replaced, not polished.
  • We need a real select tool plus a real insert/shape tool.
  • Primitive picking should lead straight into viewport placement.
  • This should also connect naturally to a future context menu and insert panel.
